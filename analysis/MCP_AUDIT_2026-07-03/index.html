    MCP Audit Report — LEVIATHAN Platform

    Date: 2026-07-03 Analyst: MCP Architect Duration: 2 hours

    Executive Summary

    Платформа LEVIATHAN содержит 4 MCP-сервера (3 активных, 1 инструментальный) и 2 MCP-клиентских навыка в Hermes Agent. Общий пул — ~50 инструментов, охватывающих файловую систему, сеть, системное администрирование, дизайн, базы данных, Android/Termux и торренты. Основной сервер — leviathan_mcp (FastMCP, HTTP:8300).

    5. Findings & Issues

    Critical

    1. Hardcoded GitHub token in config — /root/.cursor/mcp.json contains <REDACTED> in plain text.
    2. Risk: Token exposure if config is committed or shared

    3. Fix: Use environment variable or GitHub CLI auth

    4. MCP server runs as root — leviathan_mcp.service runs as root with full system access.

    5. Risk: Any tool exploit gives complete system compromise

    6. Fix: Create dedicated mcp user with limited permissions

    7. External MCP access with hardcoded Bearer token — /opt/pelleto-ai/.mcp.json uses static token den4ik1985!.

    8. Risk: Token is simple, shared in config file, and used over HTTP (not HTTPS)
    9. Fix: Use HTTPS, rotate token, use env variable

    Moderate

    1. Two competing MCP server implementations — leviathan_mcp_server.py (FastMCP, modern) and leviathan_mcp.py (vanilla MCP, legacy) both serve similar tools.
    2. Issue: Code duplication, potential inconsistency

    3. Fix: Deprecate leviathan_mcp.py and migrate all clients to the FastMCP version

    4. No rate limiting — Any MCP client can call lev_bash or systemctl without restrictions.

    5. Fix: Add rate limiting middleware

    6. SQLite-only database support — No PostgreSQL, MySQL, or MongoDB MCP tools exist.

    7. Fix: Add database query tools for active databases (PostgreSQL port 5434)

    Minor

    1. Pelleto external MCP access via public IP — 78.17.24.96:8300 is directly exposed.

    2. Fix: Use WireGuard/VPN tunnel instead of public exposure

    3. No MCP discovery documentation — New clients need to guess the MCP endpoint and auth.

    4. Fix: Add MCP discovery docs to the lev-cli toolset

    5. Context Keeper SQLite WAL — /opt/leviathan_engine/logs/lev_context.db grows unbounded.

    6. Fix: Add WAL checkpoint cron job
