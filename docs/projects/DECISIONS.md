# 🏛️ LEVIATHAN Architectural Decision Records

## ADR-001: Monorepo Architecture

**Date:** 2025-01-15 | **Status:** ✅ Accepted

**Context:** Multiple independent projects on a single server need unified management.

**Decision:** Use a single-server monorepo architecture with independent systemd services.

**Consequences:**
- ✅ Simple deployment
- ❌ No horizontal scaling
- ✅ Easy monitoring

## ADR-002: AI-First Design

**Date:** 2025-06-01 | **Status:** ✅ Accepted

**Context:** Platform needs AI capabilities for all services.

**Decision:** Claude AI as primary LLM, Gemini as fallback, MCP protocol for tool exposure.

**Consequences:**
- ✅ Rich AI capabilities
- ❌ API cost dependency
- ✅ MCP enables tool interoperability

## ADR-003: PostgreSQL Primary Database

**Date:** 2025-03-10 | **Status:** ✅ Accepted

**Context:** Multiple services need persistent storage.

**Decision:** PostgreSQL 16 as primary database, port 5434 (non-standard).

**Consequences:**
- ✅ ACID compliance
- ✅ Single management point
- ❌ Single point of failure (no replication)

## ADR-004: MCP Protocol for Tool Exposure

**Date:** 2025-07-01 | **Status:** ✅ Accepted

**Context:** AI agents need standardized access to system tools.

**Decision:** Implement Model Context Protocol (MCP) via FastMCP on port 8300.

**Consequences:**
- ✅ Standardized tool interface
- ✅ Cursor/Claude Desktop compatibility
- ❌ Root-level access security concern

## ADR-005: No Containerization (Bare Metal)

**Date:** 2025-01-15 | **Status:** ✅ Accepted

**Context:** Initial deployment needed speed over isolation.

**Decision:** Deploy all services as bare systemd units without Docker.

**Consequences:**
- ✅ Fast development
- ❌ No isolation between services
- ❌ Harder dependency management

## ADR-006: Single Server (No Cluster)

**Date:** 2025-01-15 | **Status:** ✅ Accepted

**Context:** Budget constraints limited infrastructure.

**Decision:** All 58 projects run on a single server (78.17.24.96).

**Consequences:**
- ✅ Simple management
- ❌ No high availability
- ❌ Resource contention risk
