## ⚠️ Сначала важное: какое именно «Freebuff»

«Freebuff» в заголовке задания = название tmux-сессий (`freebuff-0..6` → `freeb-7..23`), в которых 1–2 июля шла работа над `leviathan_engine/agent_service`. **Это не про systemd-сервис `freebuffd`** — отдельный установленный продукт (AES-256-KW, signed checkpoints), у него была своя авария с zombie-процессами 3 июля, уже разобрана отдельно. `SESSION_CONTEXT_2026-07-02.md` сам прямо предупреждает об этой путанице — дальше в отчёте «Freebuff» = только название рабочей сессии.

## Соответствие роадмапу

**✅ В норме:** `PRIORITY_PLAN_2026-07-03.md` корректно унаследовал открытые вопросы из `SESSION_CONTEXT_2026-07-02.md` (коммит memory_system.py, вопрос про merge в main) — цепоцка сессий контекст не теряла.

**⚠️ Отклонение 1 — формулировка «Фаза 1 не начата» устарела:** и `NEXT_SESSION_PROMPT.md`, и `SESSION_CONTEXT_2026-07-02.md` утверждают, что «ADR-002 фаза 1 — только спланирована». Фактически это уже неверно: `core_bridge/scenario_loader.py` (118 строк, класс `ScenarioPolicy`, пометка «ADR-002 Core Kernel Component 1/4») существует на диске, не закоммичен. Кто-то уже начал кодить фазу 1, минуя чекпоинт.

**⚠️ Отклонение 2 — решение P2 (LAP vs ADR-002) рискует быть обойдёно фактом:** пока Денис/Максим не высказались, на диске уже накопился незакоммиченный код, который де-факто реализует вариант б (ADR-002-путь) — не вариант а (LAP). Формально «ничего не начато без решения», по факту — начато. Стоит явно сообщить это Денису/Максиму: откладывать решение дальше — не нейтрально.

**🔴 Отклонение 3 — незакоммиченный код содержит блокирующий баг** (подробно — в `AUDIT_core_py_v4.md`): `LeviathanAgent()` не может быть сконструирован — падает с `TypeError` при каждой попытке. Продакшн-сервис держится на старой версии в памяти, но не переживёт следующий рестарт. Это не техдолг на будущее — это бомба с таймером, независимо от исхода P2.

## Архитектурная коррекция — рекомендации

1. Зафиксировать текущее незакоммиченное состояние (сняток-коммит в отдельную ветку, Не в main/feature) — чтобы не потерять core_bridge/scenario_loader.py + изменения в kernel.py/providers.py/tools.py при случайном `git checkout`/сбое.
2. Не рестартовать `leviathan_agent.service`, пока не исправлен баг конструктора (или пока не откатили core.py к `.v31`) — сейчас это гарантированный краш на старте.
3. При передаче решения P2 Денису/Максиму — упомянуть, что ожидание уже не нейтрально — код двигается в сторону ADR-002 сам по себе.
