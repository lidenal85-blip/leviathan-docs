    ЗАДАЧА: ПОЭТАПНАЯ МИГРАЦИЯ АГЕНТНОЙ СИСТЕМЫ (4.5 → 5.0 → 5.1)

    ⚠️ ГЛАВНОЕ ПРАВИЛО

    НЕ ПРЫГАЙ ЧЕРЕЗ ЭТАПЫ. Если не реализован базовый компонент версии X, НЕ переходи к версии X+1. Сначала проверь наличие кода, затем реализуй недостающее, протестируй, и только потом двигайся дальше.

    КОНТЕКСТ

    Сервер: 78.17.24.96 Проект: /opt/leviathan_engine/agent_service/ Текущее состояние: Хаотичный код (~25K строк), частично работающий, без чёткой архитектуры. Цель: Привести систему в соответствие с Манифестом через 3 фазы.

    ФАЗА 0: АУДИТ И БАЗОВАЯ ГИГИЕНА (Фундамент для 4.5)

    Цель: Подготовить код к рефакторингу. Без этого нельзя строить ничего нового.

    Задачи:

    1. Очистка мусора:
    2. Удалить все .bak, .old, fix_*.py файлы.
    3. Удалить дубликаты кода (найти одинаковые функции в разных файлах).

    4. Разбить монолитные файлы (если есть файлы >500 строк без структуры).

    5. Стандартизация импортов:

    6. Привести все импорты к единому стилю (абсолютные пути от agent_service).

    7. Добавить type hints туда, где их нет (особенно в core.py, dispatcher.py).

    8. Базовые тесты (Smoke Tests):

    9. Написать скрипт, который просто запускает агента с минимальным запросом и проверяет, что он не падает с ошибкой импорта или синтаксиса.

    Критерий перехода: Код чистый, импорты работают, базовый запуск успешен. Отчёт: /root/analysis/PHASE_0_AUDIT.md

    ФАЗА 1: РЕАЛИЗАЦИЯ МАНИФЕСТА 4.5 (Базовая надёжность)

    Цель: Сделать систему устойчивой и предсказуемой.

    Компоненты 4.5 (Обязательные):

    1. Structured Logging (Структурированное логирование):
    2. Внедрить JSON-логгер вместо print/stdout.
    3. Логи должны содержать: timestamp, level, agent_id, event_type, payload.

    4. Файл: /opt/leviathan_engine/agent_service/utils/logger.py.

    5. Error Handling & Retry (Обработка ошибок):

    6. Единый механизм обработки исключений (try-except обёртки).
    7. Реализовать экспоненциальный backoff для сетевых запросов (уже есть backoff_scheduler, нужно унифицировать).

    8. Файл: /opt/leviathan_engine/agent_service/utils/errors.py.

    9. Basic Configuration Management:

    10. Централизовать конфиги (убрать хардкод параметров).
    11. Использовать Pydantic Settings для валидации конфигов при старте.

    12. Файл: /opt/leviathan_engine/agent_service/config/settings.py.

    13. Idempotency Keys (Идемпотентность):

    14. Убедиться, что OperationRegistry работает корректно для всех внешних вызовов.
    15. Добавить генерацию ключей для каждого действия агента.

    Критерий перехода: Агент не падает при сетевых ошибках, логи структурированы, конфиги валидируются. Отчёт: /root/analysis/PHASE_1_MANIFEST_4.5.md

    ФАЗА 2: РЕАЛИЗАЦИЯ МАНИФЕСТА 5.0 (Архитектурное ядро)

    Цель: Внедрить формальную архитектуру. Только после завершения Фазы 1.

    Компоненты 5.0 (Обязательные):

    1. Agent State Machine (FSM):
    2. Реализовать класс StateMachine с состояниями: INIT, PLAN, ACT, OBSERVE, COMPLETE, FAILED.
    3. Заменить хаотичные флаги в dispatcher.py на явные переходы состояний.

    4. Файл: /opt/leviathan_engine/agent_service/core/state_machine.py.

    5. Typed State (Pydantic Models):

    6. Описать состояние агента (AgentState) как Pydantic модель.

    7. Все данные, передаваемые между модулями, должны быть типизированы.

    8. Checkpointing (Сохранение состояния):

    9. Интегрировать ContextKeeper с State Machine.
    10. Сохранять состояние при каждом переходе состояния.

    11. Реализовать возможность восстановления (resume) из последней точки.

    12. Event Bus (Событийная шина):

    13. Унифицировать существующий EventBus.
    14. Определить стандартные события: AgentStarted, TaskCompleted, ErrorOccurred.

    Критерий перехода: Агент имеет явный жизненный цикл, состояние сохраняется и восстанавливается, все данные типизированы. Отчёт: /root/analysis/PHASE_2_MANIFEST_5.0.md

    ФАЗА 3: РЕАЛИЗАЦИЯ МАНИФЕСТА 5.1 (Продвинутые возможности)

    Цель: Добавить гибкость и расширяемость. Только после завершения Фазы 2.

    Компоненты 5.1 (Обязательные):

    1. Runnable Interface:
    2. Создать абстрактный интерфейс Runnable для Agent, Tools, Chains.

    3. Реализовать композицию (| оператор) для цепочек действий.

    4. Middleware Pipeline:

    5. Внедрить систему хуков (before/after model, before/after tool).

    6. Реализовать базовые мидлвари: Logging, RateLimit, Security.

    7. Workflow Engine:

    8. Добавить поддержку ветвлений (Branch/Merge) и параллельного выполнения.

    9. Интегрировать с Checkpointing для долгих воркфлоу.

    10. Advanced Memory:

    11. Разделить память на Short-term, Long-term, Working.
    12. Добавить механизмы забывания (eviction) и сжатия контекста.

    Критерий перехода: Система поддерживает сложные сценарии, легко расширяется новыми инструментами и мидлварями. Отчёт: /root/analysis/PHASE_3_MANIFEST_5.1.md
